## 🎯 프롬프트 작성 원칙

1. **명확한 목표**: 각 프롬프트는 명확한 목표를 가짐
2. **컨텍스트 제공**: 이전 작업과의 연관성 명시
3. **구체적인 요구사항**: 폴더 구조, 파일 형식 등 구체적으로 명시
4. **역할 기반 접근**: Agent Team 시뮬레이션을 통한 다양한 관점 확보

## 💡 학습 포인트

### 성공 요인
1. **단계적 접근**: 개념 → 기획 → 설계 → 개발 플랜 순서로 진행
2. **문서화 우선**: 코드 작성 전 충분한 문서화
3. **역할 분담**: Agent Team을 통한 다각도 검토
4. **실용적 코드**: 모든 Task에 즉시 사용 가능한 코드 포함

### 개선 포인트
1. 프롬프트 재사용성을 위한 템플릿화
2. 더 구체적인 제약 조건 명시
3. 예상 결과물 예시 제공

## 📝 프롬프트 작성 팁

### 좋은 예
```
"@docs\ 문서 기반으로 @docs\ 에 4.DevelopPlan 폴더 생성해서
UI 및 기능에 대한 개발 플랜 작성해줘. 개발 플랜은 Task 별로
분리해서 작성해주고, 각 Task 별 진행 상황을 확인할 수 있는
Manager 또는 Check 파일 생성해줘"
```

**포인트**:
- ✅ 기존 문서 참조 (@docs\)
- ✅ 명확한 폴더 구조 (4.DevelopPlan)
- ✅ 구체적인 요구사항 (Task 별 분리, Manager 파일)

### 나쁜 예
```
"개발 플랜 만들어줘"
```

**문제점**:
- ❌ 너무 추상적
- ❌ 출력 형식 불명확
- ❌ 컨텍스트 부족
