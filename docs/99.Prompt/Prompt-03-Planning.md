# Prompt 03: 기획 및 설계

**날짜**: 2026-02-23
**단계**: 팀 시뮬레이션 & 상세 기획
**목적**: 다양한 역할의 관점에서 프로젝트 기획 및 설계

---

## 📝 원본 프롬프트

```
이제 Agent Team을 생성해서 이 프로젝트를 기획 및 설계를 진행할거야.
[UI/UX 디자이너, 디자이너, 프론트엔드 개발자, PM, PL] 5명이서 회의를 진행해줘.
회의에 대한 내용은 @docs/ 폴더에 '1.Plan' 폴더 생성해서 회의록 및
기획 설계서를 카테고리별로 정리해줘
```

---

## 🎯 의도

1. **다각도 검토**
   - 5가지 역할의 관점에서 프로젝트 분석
   - PM, PL, Frontend, UI/UX 디자이너, 디자이너

2. **현실적인 기획**
   - 실제 팀 회의를 시뮬레이션
   - 역할별 전문성 반영

3. **체계적 문서화**
   - 회의록 형식으로 정리
   - 카테고리별 기획서 작성

---

## ✅ 결과물

### Agent Team 구성

| 역할 | 이름 | 담당 |
|------|------|------|
| **PM** | 이서연 | 프로젝트 전체 관리, 일정/범위 |
| **PL** | 김태현 | 기술 리드, 아키텍처 설계 |
| **Frontend** | 정하은 | 프론트엔드 개발 |
| **UI/UX 디자이너** | 박지훈 | 사용자 경험 설계 |
| **디자이너** | 최예린 | 비주얼 디자인, 컬러 시스템 |

### 생성된 문서 (9개)

```
docs/
└── 1.Plan/
    ├── README.md
    ├── 01-Kickoff-Meeting.md         (회의록)
    ├── 02-Feature-Priority.md        (우선순위)
    ├── 03-User-Research.md           (페르소나 3개)
    ├── 04-UX-Design-Plan.md          (UX 설계)
    ├── 05-UI-Design-System.md        (디자인 시스템)
    ├── 06-Frontend-Architecture.md   (아키텍처)
    ├── 07-Development-Schedule.md    (4주 일정)
    └── 08-Technical-Decisions.md     (10개 ADR)
```

### 주요 결정 사항

1. **MVP 범위**: P0 우선순위 7개 기능
2. **일정**: 4주 (Week 1-4)
3. **페르소나**: 개발자(재우), 디자이너(지민), 주니어(수진)
4. **디자인 시스템**: Inter 폰트, 4px 그리드, shadcn/ui
5. **아키텍처**: Zustand 상태 관리, Modular 구조
6. **기술 결정**: 10개 ADR (Architecture Decision Record)

---

## 💡 학습 포인트

### 잘된 점
- ✅ 역할 명시 ([UI/UX 디자이너, ...])
- ✅ 출력 형식 지정 (회의록, 기획 설계서)
- ✅ 폴더 구조 명확화 (1.Plan)

### 특히 효과적이었던 부분
- **Agent Team 시뮬레이션**: 5가지 관점에서 검토
  - PM: 일정/범위 관리
  - PL: 기술적 실현 가능성
  - Frontend: 구현 복잡도
  - UI/UX: 사용자 경험
  - Designer: 비주얼 일관성

### 개선 가능한 점
- 회의 진행 방식 명시 (예: 각 역할별 의견 → 토론 → 결정)
- 원하는 문서 개수나 분량 제시
- 특정 논의 주제 제시 (예: "MVP 범위 결정", "기술 스택 최종 확정")

- **라인 수**: 약 1,500+ 라인

- **일정 계획**: 4주 상세 일정

## 🎨 프롬프트 개선 버전

더 구체적으로 작성한다면:

```
Agent Team을 구성해서 킥오프 미팅을 진행해줘.

**팀 구성**:
- PM: 프로젝트 관리, 우선순위 결정
- PL: 기술 리드, 아키텍처 설계
- Frontend 개발자: 구현 가능성 검토
- UI/UX 디자이너: 사용자 경험 설계
- 디자이너: 비주얼 디자인

**회의 안건**:
1. MVP 범위 결정 (P0/P1/P2 우선순위)
2. 4주 개발 일정 수립
3. 기술 스택 최종 확정
4. 디자인 시스템 정의

**결과물**:
@docs/1.Plan/ 폴더에 다음 문서 작성:
- 회의록
- 기능 우선순위표
- 사용자 페르소나 (3개)
- UX/UI 설계 가이드
- 프론트엔드 아키텍처
- 개발 일정표
- 기술 의사결정 기록 (ADR)

각 역할의 전문성을 반영해서 현실적으로 작성해줘.
```
