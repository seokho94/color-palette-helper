## 💡 학습 포인트

### 잘된 점
- ✅ 간결한 지시 ("카테고리별로 분리")
- ✅ 명확한 경로 지정 (`@docs\`, `0.Concept`)
- ✅ 이전 컨텍스트 활용 ("지금 작성한 내용")

### 개선 가능한 점
- 원하는 파일 개수나 분류 기준 명시
- 문서 포맷 스타일 가이드 제공 (예: 목차 포함, 이모지 사용 등)

## 🎨 프롬프트 개선 버전

더 구체적으로 작성한다면:

```
지금 작성한 내용을 다음 기준으로 분류해서 @docs\0.Concept\ 폴더에 정리해줘:

1. 프로젝트 개요 (목적, 타겟 사용자)
2. 핵심 기능 목록 (카테고리별 그룹화)
3. 기술 스택 (Frontend, Backend, 라이브러리)
4. 필요한 데이터/정보
5. 개발 로드맵
6. 참고 자료

각 문서는 다음 형식으로:
- 제목 (# Title)
- 개요
- 상세 내용
- 참고 문서 링크
```
