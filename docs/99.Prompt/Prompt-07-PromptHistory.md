## 🎨 프롬프트 작성 베스트 프랙티스

### 1. 명확한 목표 설정
**나쁜 예**: "문서 만들어줘"
**좋은 예**: "@docs\4.DevelopPlan 폴더에 Task 별로 개발 플랜 작성해줘"

### 2. 컨텍스트 제공
**나쁜 예**: "기획서 작성해줘"
**좋은 예**: "@docs\ 문서를 기반으로 기획서 작성해줘"

### 3. 출력 형식 명시
**나쁜 예**: "정리해줘"
**좋은 예**: "카테고리별로 분리해서 md 파일로 정리해줘"

### 4. 역할/팀 활용
**나쁜 예**: "기능 정의서 작성해줘"
**좋은 예**: "Agent Team을 [PM, PL, Frontend, ...] 로 구성해서 기능 정의서 작성해줘"

### 5. 예시 제공
**나쁜 예**: "문서 구조 만들어줘"
**좋은 예**: "다음 구조로 문서 작성해줘: 1. 목표 2. 체크리스트 3. 코드 예제"

## 🔍 프롬프트 패턴 분석

### 패턴 1: 문서 생성 요청
```
[기존 문서 참조] + [출력 경로] + [분류 기준] + [출력 형식]
```
예: "@docs\ 문서 기반으로 @docs\2.Functional 에 기능 별로 md 파일 작성해줘"

### 패턴 2: 팀 시뮬레이션
```
"Agent Team을 [역할 목록] 로 구성해서 [작업] 진행해줘"
```
예: "Agent Team을 [PM, PL, Frontend, ...] 로 구성해서 기획 회의 진행해줘"

### 패턴 3: 구조화된 출력
```
[내용] + [폴더 생성] + [카테고리별 정리]
```
예: "지금 작성한 내용을 카테고리 별로 분리해서 0.Concept 폴더 생성해서 정리해줘"
