## 1. 리서치 개요

### 1.1 목적
- 타겟 사용자의 니즈 파악
- 기존 도구 사용 시 pain point 발견
- 차별화 포인트 검증

### 1.2 방법론
- **데스크 리서치**: 기존 도구(Coolors, Adobe Color) 리뷰 분석
- **경쟁 분석**: 유사 서비스 기능 비교
- **설문조사**: 디자이너/개발자 커뮤니티 (Reddit, Twitter)
- **1:1 인터뷰**: 타겟 사용자 5명 (예정)

## 2. 타겟 사용자 분석

### 2.1 사용자 세그먼트

| 세그먼트 | 비중 | 특징 | 니즈 |
|---------|------|------|------|
| **프리랜서 디자이너** | 30% | 시간 제약, 빠른 작업 필요 | 빠른 팔레트 생성, 트렌디한 조합 |
| **스타트업 개발자** | 40% | 디자이너 없음, 직접 작업 | 접근성 보장, 개발 친화적 |
| **주니어 디자이너** | 20% | 색상 이론 부족 | 가이드, 학습 자료 |
| **에이전시 디자이너** | 10% | 클라이언트 요구 다양 | 산업별 레퍼런스 |

### 2.2 사용자 여정 (Current State)

```
[새 프로젝트 시작]
  ↓
1. 영감 찾기
   - Dribbble, Pinterest 검색 (30분~1시간)
   - 경쟁사 웹사이트 분석
   ❌ Pain: 시간 소모, 일관성 없음
  ↓
2. 색상 선택
   - Coolors로 팔레트 생성 (반복 시행착오)
   - Adobe Color로 조화 확인
   ❌ Pain: 접근성 확인 따로 해야 함
  ↓
3. 검증
   - WebAIM 대비 계산기로 수동 체크
   - 실제 UI에 적용해서 확인
   ❌ Pain: 반복 작업, 비효율적
  ↓
4. 적용
   - Figma/코드에 수동 입력
   ❌ Pain: 복사 붙여넣기 반복
```

**총 소요 시간**: 1~2시간

## 3. 경쟁 서비스 분석

### 3.1 주요 Pain Points

| 도구 | 강점 | 약점 | 기회 |
|------|------|------|------|
| **Coolors** | 빠른 생성, 직관적 | 접근성 검증 약함 | ✅ 우리가 보완 |
| **Adobe Color** | 이론 탄탄 | 복잡한 UI, 무거움 | ✅ 간결한 UX |
| **Paletton** | 정확한 조화 | 오래된 디자인 | ✅ 모던한 UI |
| **WebAIM** | 접근성 전문 | 팔레트 생성 없음 | ✅ 통합 제공 |

### 3.2 사용자 불만 사항 (Reddit, Twitter 분석)

**접근성 관련** (42%):
> "팔레트는 예쁜데 대비 비율이 안 맞아서 다시 만들어야 함"
> "매번 WebAIM에서 확인하는 게 귀찮음"

**워크플로우** (31%):
> "여러 도구를 왔다갔다 해야 함"
> "Figma랑 연동 안 돼서 수동 입력..."

**학습 곡선** (18%):
> "어떤 조화 규칙을 써야 할지 모르겠음"
> "색상 이론을 잘 몰라도 쓸 수 있었으면"

**커스터마이징** (9%):
> "로고 색상 기반으로 팔레트 만들고 싶은데"

## 6. 핵심 인사이트

### 6.1 사용자가 진짜 원하는 것

**❌ 많은 기능**
**✅ 빠르고 정확한 하나의 워크플로우**

> "도구를 3개 쓰는 것보다 하나로 끝낼 수 있으면 좋겠어요" - 지민

### 6.2 접근성이 핵심 차별점

**기존 도구의 가장 큰 gap**:
- Coolors: 예쁜 팔레트 생성 ✅ / 접근성 검증 ❌
- WebAIM: 접근성 검증 ✅ / 팔레트 생성 ❌

**우리의 기회**:
- 두 가지를 **통합**하는 유일한 도구

### 6.3 "도구"가 아닌 "에듀케이션"

주니어 사용자(수진)에게는:
- 단순히 팔레트를 만드는 게 아니라
- 색상 이론을 **학습**하는 경험 제공

**장기 경쟁력**:
- 사용자 충성도 향상
- 커뮤니티 형성 가능

## 7. UX 설계 방향

### 7.1 핵심 원칙

1. **Zero Learning Curve**
   - 첫 방문자도 30초 안에 팔레트 생성 가능

2. **Immediate Feedback**
   - 색상 변경 → 즉시 미리보기 업데이트
   - 접근성 문제 → 즉시 경고 표시

3. **Guidance, Not Obstruction**
   - 자동 제안은 하되, 강제하지 않음
   - 사용자가 자유롭게 조정 가능

### 7.2 정보 우선순위

**화면에서 가장 눈에 띄어야 할 것**:
1. 생성된 팔레트 (색상 칩)
2. 접근성 경고 (있다면)
3. 미리보기 (실제 적용 모습)
4. 조정 컨트롤 (색상 수정)

**숨겨도 되는 것**:
- 고급 옵션 (색상 잠금, 세부 조정)
- 히스토리, 즐겨찾기 (사이드바)
