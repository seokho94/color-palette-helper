## 1. 기능 우선순위 매트릭스

### 1.1 평가 기준

| 기준 | 설명 | 가중치 |
|------|------|--------|
| **비즈니스 가치** | 사용자 확보 및 차별화에 기여하는가? | 40% |
| **기술 복잡도** | 구현 난이도 및 리스크 | 30% |
| **사용자 니즈** | 타겟 사용자가 실제로 원하는가? | 30% |

### 1.2 우선순위 분류

**P0 (Must Have)**: MVP 필수, 없으면 서비스 성립 불가
**P1 (Should Have)**: 중요하지만 Phase 2로 연기 가능
**P2 (Nice to Have)**: 있으면 좋지만 장기 계획
**P3 (Won't Have)**: 현재 범위 밖

## 5. 우선순위 변경 시나리오

### 5.1 일정이 촉박한 경우 (3주로 단축)

**제거 우선순위**:
1. URL 공유 기능 (Phase 2로 연기)
2. 색상 조화 규칙 축소 (보색, 유사색, 삼색만)
3. 폼 미리보기 제거
4. 대안 색상 제안 제거

**핵심만 남김**:
- 색상 입력 (Picker + 이미지)
- 3가지 조화 규칙
- 접근성 검증 (경고만)
- 버튼/카드 미리보기
- 코드 복사

### 5.2 사용자 피드백으로 우선순위 변경

**만약 "URL 분석" 요청이 압도적으로 많다면**:
- Week 3에 간단한 백엔드 추가
- Puppeteer로 스크린샷 구현
- 다른 기능(히스토리, 즐겨찾기) 연기

## 7. 의사결정 기록

### 7.1 왜 URL 분석을 MVP에서 제외했나?

**찬성 의견 (PM, Designer)**:
- 경쟁 서비스와의 차별점
- 디자이너들이 자주 쓸 기능

**반대 의견 (PL, Frontend)**:
- 백엔드 서버 필요 (Puppeteer)
- 배포 복잡도 증가 (Vercel Functions 제약)
- CORS, 성능 이슈

**최종 결정**: Phase 2로 연기
- **이유**: 기술 리스크 > 비즈니스 가치 (MVP 단계)
- **대안**: 이미지 업로드로 유사한 경험 제공

### 7.2 왜 AI 추천을 포함하지 않았나?

**고려 사항**:
- OpenAI API 비용 (무료 티어 제한)
- 정확도 보장 어려움
- 데이터 수집 필요

**최종 결정**: Phase 5로 연기
- **이유**: 규칙 기반으로도 충분한 가치 제공 가능
- **대안**: 수동으로 큐레이션한 산업/무드 팔레트 (Phase 2)
