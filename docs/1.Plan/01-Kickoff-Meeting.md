## 2. 프로젝트 개요 발표 (PM 김민수)

### 2.1 배경
- 웹 서비스 개발 시 색상 선택은 브랜딩과 사용자 경험에 중요
- 기존 도구들은 단순 팔레트 생성에 그침
- **접근성 검증 + 산업별 추천**을 결합한 차별화된 서비스 필요

### 2.2 비즈니스 목표
1. **1차 목표**: MVP 4주 내 출시 → 얼리어답터 확보
2. **2차 목표**: 사용자 피드백 기반 기능 확장
3. **3차 목표**: Figma 플러그인 연동으로 디자이너 생태계 진입

### 2.3 성공 지표
- 출시 1개월 내 **MAU 1,000명**
- **평균 세션 5분 이상** (충분한 사용성)
- **팔레트 생성 완료율 70% 이상**

## 3. 기술 아키텍처 제안 (PL 이서연)

### 3.1 기술 스택 제안
```
Frontend: Next.js 14 (App Router) + TypeScript
Styling: TailwindCSS + shadcn/ui
State: Zustand (경량 전역 상태)
Color: chroma.js (색상 조작) + color-thief (추출)
Deploy: Vercel (CI/CD 자동화)
```

**선정 이유**:
- Next.js: SEO 최적화 + SSR 지원
- TailwindCSS: 빠른 프로토타이핑
- Vercel: 무료 호스팅 + 자동 배포

### 3.2 아키텍처 원칙
1. **컴포넌트 재사용성**: 공통 UI 컴포넌트 라이브러리 구축
2. **성능 우선**: 색상 계산 최적화 (Web Worker 고려)
3. **확장 가능성**: API 분리 구조 (향후 백엔드 추가 용이)

### 3.3 우려사항
- **이미지 처리 성능**: 큰 이미지 업로드 시 브라우저 렌더링 블록
  - **해결안**: Web Worker로 비동기 처리

## 4. UX 설계 방향 (UI/UX Designer 박지훈)

### 4.1 사용자 플로우 제안

```
[시작]
  ↓
1. 색상 입력 (3가지 방법)
   - Color Picker
   - 이미지 업로드
   - URL 입력
  ↓
2. 팔레트 생성
   - 조화 규칙 선택 (보색, 유사색 등)
   - 산업/무드 선택 (선택사항)
  ↓
3. 미리보기 및 조정
   - UI 컴포넌트에 적용된 모습 확인
   - 접근성 검증 결과 확인
   - 색상 미세 조정
  ↓
4. 내보내기
   - 코드 복사 (HEX, RGB, CSS)
   - 이미지 다운로드 (선택사항)
```

### 4.2 핵심 UX 원칙
1. **즉각적 피드백**: 색상 변경 시 실시간 미리보기
2. **명확한 안내**: 접근성 기준 미달 시 경고 + 개선안 제시
3. **효율적인 워크플로우**: 3번의 클릭 내 팔레트 완성

### 4.3 제안사항
- **첫 방문자를 위한 튜토리얼** (인터랙티브 가이드)
- **키보드 단축키 지원** (Spacebar: 재생성)

## 5. 디자인 시스템 방향 (Designer 최예린)

### 5.1 비주얼 컨셉
**"클린하고 전문적인 도구"**

- **컬러 팔레트**:
  - 베이스: 모노톤 (Black, White, Gray)
  - 액센트: 사용자가 생성한 색상이 주인공
  - 예시: `#000000`, `#FFFFFF`, `#F5F5F5`, `#E5E5E5`

- **타이포그래피**:
  - Heading: Inter (명확하고 모던)
  - Body: Inter (일관성)
  - Code: JetBrains Mono (개발자 친화적)

- **레이아웃**:
  - 3단 구조: `[입력] [팔레트] [미리보기]`
  - 데스크톱 우선 설계

### 5.2 디자인 시스템 구성 요소
1. **Color Variables**: CSS Custom Properties
2. **Spacing Scale**: 4px 기반 (4, 8, 12, 16, 24, 32...)
3. **Border Radius**: 4px (버튼, 카드), 8px (패널)
4. **Shadow**: 3단계 (sm, md, lg)

### 5.3 참고 디자인
- Figma, Linear (미니멀 전문성)
- Coolors.co (색상 도구)

## 6. 기술 구현 검토 (Frontend Developer 정하은)

### 6.1 핵심 라이브러리 검증

| 라이브러리 | 용도 | 검증 결과 |
|-----------|------|----------|
| chroma.js | 색상 조작 | ✅ 가벼움 (13KB), API 직관적 |
| color-thief | 이미지 색상 추출 | ✅ 브라우저 호환성 좋음 |
| react-colorful | Color Picker | ✅ 접근성 우수, 커스터마이징 가능 |

### 6.2 성능 고려사항
1. **색상 계산 최적화**:
   - 조화 규칙 계산 → Memoization
   - 대량 색상 생성 → Debounce/Throttle

2. **이미지 처리**:
   - 최대 크기 제한: 5MB
   - Canvas API 활용 → 리사이징

3. **번들 사이즈**:
   - 초기 로드 < 200KB (gzip)
   - Code Splitting (이미지 분석 기능)

### 6.3 개발 환경 제안
```bash
Node.js: v20 LTS
Package Manager: pnpm (빠른 설치)
Linting: ESLint + Prettier
Testing: Vitest (단위 테스트)
```

### 6.4 우려사항 및 해결
- **문제**: URL 스크린샷 (브라우저 제약)
- **해결**: MVP에서는 제외 → Phase 2에서 백엔드 추가

### 7.1 MVP 범위 최종 확정

**✅ 포함**
1. 색상 입력 (Color Picker, 이미지 업로드)

- AI 기반 추천
